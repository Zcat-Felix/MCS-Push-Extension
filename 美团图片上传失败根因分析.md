# 美团图片上传：调试能显示 vs 实际测试失败 — 根因分析

## TL;DR
**调试脚本绕过 Python `.format()` 直接发送 JS → 成功。fill_engine.py 的 JS 经 `.format(tid=...)` 处理时, 外层 `try{` 单花括号被误认为替换字段名 → `ValueError` 崩溃 → `_cdp_eval` 返回空字符串 → `_fill_images` 判定 10/10 张上传失败。**

---

## 一、差异点列表

### 差异 #1 (🔴 致命): JS 字符串生成路径

| | 调试脚本 (debug_upload.py) | fill_engine.py (生产路径) |
|---|---|---|
| JS 构建方式 | Python 字符串直接拼接, 硬编码 `temp_id` | Python 隐式字符串拼接 + `.format(tid=temp_id)` |
| 花括号处理 | 不经过 `.format()`, 所有 `{` `}` 按字面值传递 | 必须遵守 Python `.format()` 语法: `{tid}` 是替换字段, 其余所有 `{` `}` 必须 `{{` `}}` 双写 |
| 实际代码行 | `"uv.processAndUploadFile(inp.files[0]).then(...);return'started:0'"` | `"try{"` → `.format()` 看到单 `{` → `ValueError` |
| 结果 | ✅ JS 正确生成并送达 CDP | ❌ 在 Python 进程内 `.format()` 崩溃, JS 从未发送到浏览器 |

### 差异 #2 (🟡 协作): _cdp_eval 错误吞没

| | 旧版 _cdp_eval | 新版 _cdp_eval (v3.2) |
|---|---|---|
| 错误处理 | `json.loads(r.stdout).get('value', '')` — 丢弃 `error` 字段 | 检测 `'error' in resp` → 返回 `'CDP_EVAL_ERR:...'` |
| 结果 | `ValueError` 被外层 `except` 捕获 → 返回 `''` → 无法区分"JS 错误"和"Python 崩溃" | 明确返回 `CDP_EVAL_ERR:` 前缀, 可被日志捕获 |

### 差异 #3 (🟡 残留): 系统资源状态

| 维度 | 调试环境 | fill_engine 生产环境 |
|---|---|---|
| 运行进程 | 主进程直接 `python debug_upload.py` | MCP Server 子进程 `spawn(python, fill_engine.py)` |
| 环境变量 | 当前 shell 环境 | `{...process.env, PYTHONIOENCODING:'utf-8', PYTHONUNBUFFERED:'1'}` |
| CDP 连接 | 手动 curl 已验证 | 子进程 curl 需重新建立连接 |
| 页面状态 | 已知的稳定状态 (已清理空占位) | 前次残留状态 (`valueSelf:[{src:"",...}]`) |

### 差异 #4 (🟡): 标签切换逻辑

| | 旧版 | 新版 (v3.2) |
|---|---|---|
| 查找方式 | `Array.from(tabs).find(t => t.textContent.trim()==='本地上传')` | `tabs[1]` (直接索引) |
| 问题 | CDP eval 中 `for` 循环 + 中文比较 → 空返回 | ✅ 无中文 token |
| 回退机制 | CDP 物理点击 `Input.dispatchMouseEvent` → 坐标计算复杂 | `mouseenter` + `mousedown` + `click()` 三事件 |

### 差异 #5 (🟡): 组件状态清理

| | 旧版 | 新版 (v3.2) |
|---|---|---|
| Pre-clean | 无 | Step 1 前清理 `valueSelf` 空占位 |
| Step 5 清理 | `pv.valueSelf = validItems` (数组直接赋值) | `pv.valueSelf.splice(0, len)` + `push()` (Vue2 响应式安全) |
| 父组件同步 | 只写子组件 | 分别写入 `ProductPictureV3.value` + `.product-picture-add.valueSelf` |

## 三、最终根因结论

### 🔴 根因 #1 (致命): Python `.format()` 花括号碰撞

```
代码行: "try{"                            ← Python 字符串

Python .format() 看到 try 后面的 { :
  ├── 期望 {field_name} 或 {{escaped}}
  ├── 实际遇到: {var inp=document.getElementById('_mt_upload_tmp');...}
  └── 结果: ValueError('unexpected { in field name')
```

**完整错误链**:
1. `_fill_images` 构造 JS 字符串: 外层 `try{` (单 `{`) + 内层 `try{{uv.process...}}` (双 `{{}}`)
2. `.format(tid=temp_id)` 扫描字符串: 找到 `{tid}` 替换 → 然后遇到 `try{` → 单 `{` 不是 `{{` → 触发替换字段解析
3. `{var inp=document.getElementById('_mt_upload_tmp');...` 不是合法的字段名 → `ValueError`
4. 异常传播到 `fill_engine.py` 顶层 `except` → 整个任务标记为失败
5. `_cdp_eval` 旧版只取 `value` 关键字, 丢弃 `error` → 返回空字符串 → 无法诊断

**为什么调试能成功?**
- `debug_upload.py` 中的 JS 是用 Python 硬编码字符串: `"uv.processAndUploadFile(inp.files[0]).then(...);return'started:0'"`
- 没有经过 `.format()`, 花括号按字面值传递 → JS 正确生成 → CDP 正常执行

**为什么实际测试失败?**
- `meituan_flash.py` 中的 JS 经过 `.format(tid=temp_id)`: `"try{"` 单花括号触发 Python 格式解析
- `ValueError` 在 Python 进程内崩溃 → JS 从未送达浏览器
- 10 个文件, 每个重试 3 次, 全部在 `.format()` 阶段崩溃 → 0/10 上传成功

### 🟡 根因 #2 (次要): 错误信息丢失
- 旧版 `_cdp_eval` 只取响应中的 `value` 字段, 丢弃 `error` 字段
- MCP Server 返回 `{"error": "Uncaught"}` 时, `_cdp_eval` 返回 `''`
- `_fill_images` 接收到空字符串 → 判定为"start failed" → 无有效诊断信息

### 修复对照表
| 文件 | 行 | 修复前 | 修复后 |
|------|-----|--------|--------|
| meituan_flash.py | ~1128 | `"try{"` | `"try{{"` |
| meituan_flash.py | ~1147 | `"}}catch(e){{...}}"` | `"}}catch(e){{...}}"` (已正确) |
| meituan_flash.py | ~1148 | `"}catch(e){{..."` | `"}}catch(e){{..."` |
| meituan_flash.py | ~15-30 | `json.loads(r.stdout).get('value','')` | 增加 `error` 字段检测 |
| meituan_flash.py | ~1023 | 无 pre-clean | 打开弹窗前清理 `valueSelf` |
| meituan_flash.py | ~1229-1251 | `pv.valueSelf = validItems` | `splice(0, len)` + `push()` |
| meituan_flash.py | ~1036 | `Array.find + '本地上传'` | `tabs[1]` 索引 |

### 经验教训
1. **Python `.format()` 和 JavaScript `{}` 天然冲突**: 任何同时包含 `.format(tid=...)` 和 JS 花括号的代码, 必须用 `{{}}` 双写所有非替换字段的花括号 (Rule #24)
2. **调试环境 ≠ 生产环境**: 手动 CDP eval 绕过 Python 字符串处理 → 验证了 JS 逻辑正确, 但不代表 Python 代码能正确生成该 JS
3. **错误吞没是排障天敌**: `_cdp_eval` 丢弃 `error` 导致 `ValueError` 被静默转化为空字符串, 追查成本极高
4. **format() 是高风险 API**: 在包含 JS/CSS/正则等大量花括号的上下文中, 优先用 `string.replace('{tid}', tid)` 或 f-string 替代 `.format()`
